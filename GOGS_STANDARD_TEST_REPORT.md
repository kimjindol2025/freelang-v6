## 🔍 상세 검증 내용

### TC-01: 변수 선언 및 할당 (The Variable Test)

**목적**: 렉서와 파서가 변수를 정확히 인식하고, 세미콜론 처리가 정상화되었는지 확인

**테스트 코드** (`01_variable.free`):
```freelang
let a = 10
let b = 20
let sum = a + b
println(sum)
```

**검증 포인트**:
- ✅ 세미콜론 없이도 정상 작동 (v4 문제 해결됨)
- ✅ 변수 할당 및 접근 정상
- ✅ 산술 연산 (+) 작동

**실제 출력**: `30`
**기대값**: `30` (10 + 20)
**상태**: ✅ **통과**

---

### TC-02: 산술 연산 (The Arithmetic Test)

**목적**: 연산자 우선순위(PEMDAS)를 올바르게 처리하는지 확인

**테스트 코드** (`02_math.free`):
```freelang
let result = 10 + 5 * 2
println(result)
```

**검증 포인트**:
- ✅ 곱셈이 덧셈보다 먼저 실행됨
- ✅ 왼쪽에서 오른쪽 계산 순서 유지
- ✅ 컴파일러의 연산자 우선순위 정확

**계산 프로세스**:
```
10 + 5 * 2
= 10 + (5 * 2)   ← 곱셈 우선
= 10 + 10
= 20
```

**실제 출력**: `20`
**기대값**: `20` (NOT 30)
**상태**: ✅ **통과** - **우선순위 정확 증명**

---

### TC-03: 조건문 (The Logic Test)

**목적**: 프로그램 흐름 제어 및 블록 문법이 정상 작동하는지 확인

**테스트 코드** (`03_if_statement.free`):
```freelang
let score = 85
if (score > 80) {
  println("Pass")
} else {
  println("Fail")
}
```

**검증 포인트**:
- ✅ 비교 연산자 (>) 작동
- ✅ 중괄호 ({}) 블록 파싱
- ✅ if-else 조건 분기
- ✅ 변수 참조 및 비교

**실행 흐름**:
```
score = 85
score > 80 ? → true
→ if 블록 실행
→ println("Pass")
```

**실제 출력**: `Pass`
**기대값**: `Pass`
**상태**: ✅ **통과** - **블록/비교연산 정상**

### 핵심 발견사항

1. **세미콜론 자동 처리**: v4에서 "Unexpected token ';'" 에러가 발생했으나, v6에서는 세미콜론이 선택사항으로 정상 처리됨
   
2. **연산자 우선순위 정확**: `10 + 5 * 2 = 20`으로 곱셈이 덧셈보다 먼저 실행되는 것을 증명

3. **블록 문법 완성**: if-else의 중괄호 블록과 비교 연산자가 충돌 없이 작동
