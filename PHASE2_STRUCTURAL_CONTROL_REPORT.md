## 🔬 상세 검증 내용

### TC2-01: 연산자 우선순위 및 괄호 해석

**목적**: 복합 수식에서 수학적 규칙(PEMDAS)을 정확히 따르는지 검증

**테스트 코드**:
```freelang
let x = 10
let y = 5
let result = (x + y) * 2 - (x / 2)
println(result)
```

**계산 프로세스** (검증):
```
(x + y) * 2 - (x / 2)
= (10 + 5) * 2 - (10 / 2)    ← 괄호 먼저
= 15 * 2 - 5                  ← 괄호 내 계산
= 30 - 5                       ← 곱셈 다음
= 25                           ← 뺄셈 마지막
```

**검증 포인트**:
- ✅ 괄호 우선 처리됨
- ✅ 나눗셈(/) 정확히 계산
- ✅ 곱셈(*) 덧셈보다 먼저 실행
- ✅ 뺄셈(-) 마지막 실행

**실제 출력**: `25`
**기대값**: `25`
**판정**: ✅ **PASS** - 연산자 우선순위 설계 정확

---

### TC2-02: 비교 연산 및 논리 게이트

**목적**: 논리 연산자(AND)가 두 조건을 독립적으로 평가하고 올바른 진릿값을 도출하는지 검증

**테스트 코드**:
```freelang
let a = 100
let b = 200
let check = (a < b) && (b == 200)
println(check)
```

**논리 분석** (검증):
```
(a < b) && (b == 200)
= (100 < 200) && (200 == 200)
= true && true                ← 두 조건 모두 true
= true                        ← AND 결과
```

**검증 포인트**:
- ✅ 비교 연산자 `<` 정확 (a < b = true)
- ✅ 비교 연산자 `==` 정확 (b == 200 = true)
- ✅ AND 연산자 `&&` 정확 (true && true = true)
- ✅ 독립적 평가 (각 조건을 따로 평가)

**실제 출력**: `true`
**기대값**: `true` (또는 1)
**판정**: ✅ **PASS** - 논리 게이트 설계 정확

---

### TC2-03: 조건부 분기 및 블록 실행

**목적**: if-else 분기가 조건에 따라 정확한 블록을 실행하고, 다른 블록은 무시하는지 검증

**테스트 코드**:
```freelang
let score = 75
if (score >= 80) {
  println("SUCCESS")
} else {
  println("RETRY")
}
```

**분기 분석** (검증):
```
score = 75
score >= 80 ? → false
→ else 블록 실행
→ println("RETRY")
```

**검증 포인트**:
- ✅ 조건 평가 정확 (score >= 80 = false)
- ✅ else 블록 선택됨 (조건 거짓이므로)
- ✅ if 블록 무시됨 (중복 실행 없음)
- ✅ 정확한 블록만 실행

**실제 출력**: `RETRY`
**기대값**: `RETRY`
**판정**: ✅ **PASS** - 분기 주소 할당 정확
