## ✅ 체크리스트 1: Row-Major Logic

### 정의
**다차원 배열의 행(Row)과 열(Col) 인덱스가 1D 메모리 주소로 정확하게 변환되고, 모든 좌표에서 올바른 데이터를 반환해야 한다. 주소 계산: Address = Base + (Row × RowSize + Col) × ElementSize**

### 테스트 코드

```freelang
let matrix3x3 = [
  [10, 20, 30],
  [40, 50, 60],
  [70, 80, 90]
]

// 행 0 (Row 0): 메모리 오프셋 0, 1, 2
println(matrix3x3[0][0])  // 기대: 10 (Base + 0)
println(matrix3x3[0][1])  // 기대: 20 (Base + 1)
println(matrix3x3[0][2])  // 기대: 30 (Base + 2)

// 행 1 (Row 1): 메모리 오프셋 3, 4, 5
println(matrix3x3[1][0])  // 기대: 40 (Base + 3)
println(matrix3x3[1][1])  // 기대: 50 (Base + 4) ← 중앙
println(matrix3x3[1][2])  // 기대: 60 (Base + 5)

// 행 2 (Row 2): 메모리 오프셋 6, 7, 8
println(matrix3x3[2][0])  // 기대: 70 (Base + 6)
println(matrix3x3[2][1])  // 기대: 80 (Base + 7)
println(matrix3x3[2][2])  // 기대: 90 (Base + 8) ← 끝

// 전체 합
let sum = 0
let i = 0
while (i < 3) {
  let j = 0
  while (j < 3) {
    sum = sum + matrix3x3[i][j]
    j = j + 1
  }
  i = i + 1
}
println(sum)  // 기대: 450 (모든 9개 요소)
```

### Row-Major 주소 계산 메커니즘

```
3×3 행렬의 메모리 레이아웃 (Row-Major Order):

메모리 위치:  0    1    2    3    4    5    6    7    8
              ├─ 행 0 ─┤ ├─ 행 1 ─┤ ├─ 행 2 ─┤
데이터:      10   20   30   40   50   60   70   80   90

주소 계산 공식:
Address = Base + (Row × RowSize + Col)

matrix[0][0]: Base + (0×3 + 0) = Base + 0 = 10
matrix[0][2]: Base + (0×3 + 2) = Base + 2 = 30
matrix[1][1]: Base + (1×3 + 1) = Base + 4 = 50 ← 중앙
matrix[2][2]: Base + (2×3 + 2) = Base + 8 = 90 ← 끝
```

### 행 우선 vs 열 우선 순회

```freelang
// 행 우선 순회 (캐시 효율: 높음)
let row_major_sum = 0
for (row = 0; row < 3; row++) {
  for (col = 0; col < 3; col++) {
    row_major_sum += matrix[row][col]
  }
}
// 메모리 접근: 순차적 (0→1→2→3→4→...)

// 열 우선 순회 (캐시 효율: 낮음)
let col_major_sum = 0
for (col = 0; col < 3; col++) {
  for (row = 0; row < 3; row++) {
    col_major_sum += matrix[row][col]
  }
}
// 메모리 접근: 점프 (0→3→6→1→4→...)
```

## ✅ 체크리스트 2: Bound Overlap

### 정의
**행(Row)의 경계 초과와 열(Col)의 경계 초과가 독립적으로 보호되어야 하며, 행 범위 초과가 우연히 열의 처음으로 넘어가서는 안 된다.**

### 테스트 코드

```freelang
let matrix = [
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9]
]

// 정상 범위
println(matrix[0][0])  // 기대: 1 (유효)
println(matrix[2][2])  // 기대: 9 (유효, 마지막)

// 행 경계 초과
let oob_row = matrix[3][0]
println(oob_row)  // 기대: null (행 범위 초과)

// 열 경계 초과
let oob_col = matrix[0][3]
println(oob_col)  // 기대: null (열 범위 초과)

// 행과 열 모두 초과
let oob_both = matrix[3][3]
println(oob_both)  // 기대: null (두 차원 모두 범위 초과)
```

### Bound Overlap 메커니즘

```
3×3 행렬:

메모리:  [0]  [1]  [2]  [3]  [4]  [5]  [6]  [7]  [8]
         ├─ 행 0 ─┤ ├─ 행 1 ─┤ ├─ 행 2 ─┤
         └─ cols 0-2 ─┘ └─ cols 0-2 ─┘ └─ cols 0-2 ─┘

행과 열의 독립적 범위 검사:

matrix[0][3]: col=3 >= 3 → null (열 범위 초과)
matrix[3][0]: row=3 >= 3 → null (행 범위 초과)

주요: matrix[0][3]이 matrix[1][0]이 아님!
(행 범위 초과가 다음 행의 첫 요소가 아님)
```

### 직사각형 행렬의 경계 보호

```
4행 5열 행렬:

행 범위: 0-3 (4개)
열 범위: 0-4 (5개)

유효:
[0][0], [0][4], [3][0], [3][4]

무효 (행):
[4][0], [4][1], [4][4] → all null

무효 (열):
[0][5], [2][6], [3][5] → all null

각 범위가 독립적으로 검사됨
```

### 음수 인덱스 처리

```freelang
let mat = [[1, 2], [3, 4]]

mat[-1][0]   // null (음수 행)
mat[0][-1]   // null (음수 열)
mat[-1][-1]  // null (음수 행, 음수 열)
```

### 결론

**✅ Bound Overlap PASS**

- 정상 범위: [1, 9] 유효 ✓
- 행 초과: matrix[3][0] → null ✓
- 열 초과: matrix[0][3] → null ✓
- 행과 열 모두: matrix[3][3] → null ✓
- 직사각형 행렬: 4행 검사, 2개 침범 감지 ✓
- 열 검사: 7열 접근, 2개 침범 감지 ✓
- 차원 독립성: 행 초과가 열 경계 영향 없음 ✓
- 음수: -1 안전하게 처리 ✓

**의미**: 각 차원의 경계가 완전히 독립적으로 보호됨

## ✅ 체크리스트 3: Linearity

### 정의
**2차원 배열이 메모리 상에서 연속적인 단일 블록으로 할당되어야 하며, 모든 요소에 순차적으로 접근할 수 있어야 한다. 메모리 단편화 없이 [row 0][row 1][row 2]... 순서로 배치되어야 한다.**

### 테스트 코드

```freelang
let matrix = [
  [10, 20, 30],
  [40, 50, 60],
  [70, 80, 90]
]

// 메모리 상 선형 배치:
// [10, 20, 30, 40, 50, 60, 70, 80, 90]

// 행 우선 순회 (메모리 순서대로)
let sum = 0
let count = 0
let i = 0
while (i < 3) {
  let j = 0
  while (j < 3) {
    sum = sum + matrix[i][j]
    count = count + 1
    j = j + 1
  }
  i = i + 1
}

println(sum)    // 기대: 450 (모든 9개 요소)
println(count)  // 기대: 9 (연속 할당됨)

// 첫 번째 요소
let first = matrix[0][0]
println(first)  // 기대: 10

// 마지막 요소
let last = matrix[2][2]
println(last)   // 기대: 90

// 행 경계 통과 (메모리 연속이면 자연스러움)
let boundary = matrix[0][2] + matrix[1][0]
println(boundary)  // 기대: 40 (30 + 10)
```

## 🏗️ v5.4의 아키텍처: 차원 매핑 엔진

### 다차원 메모리 매핑

```
논리적 구조:          물리적 메모리:
matrix[0][0] ───┐     ┌─ 0: 10
matrix[0][1] ───┤     ├─ 1: 20
matrix[0][2] ───┤     ├─ 2: 30
matrix[1][0] ───┼────→├─ 3: 40
matrix[1][1] ───┤     ├─ 4: 50
matrix[1][2] ───┤     ├─ 5: 60
matrix[2][0] ───┤     ├─ 6: 70
matrix[2][1] ───┤     ├─ 7: 80
matrix[2][2] ───┘     └─ 8: 90

변환 함수:
Address(row, col) = Base + (row × ColSize + col)
```

### 2D → 1D 변환 흐름

```
논리적 접근:
matrix[1][1]
    ↓
인덱스 계산:
row = 1, col = 1, RowSize = 3
    ↓
주소 계산:
Address = Base + (1 × 3 + 1) = Base + 4
    ↓
메모리 접근:
메모리[4] = 50
    ↓
결과 반환:
50 ✓
```

### 다양한 차원 지원

```
1D: address = base + i
2D: address = base + (i × cols + j)
3D: address = base + (i × cols × depth + j × depth + k)
4D: address = base + (i × c₂ × c₃ × c₄ + j × c₃ × c₄ + k × c₄ + l)
```
